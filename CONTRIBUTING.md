# Contributing to Cracking the FDE Interview

First off — thank you. This resource exists because people like you give back.

Whether you're fixing a typo, adding a problem, sharing a company experience, or writing an entire new section — every contribution makes this better for thousands of engineers trying to break into FDE and AI Engineering roles.

## What We Need Most

- **Company experiences** — Interview reports from Google FDE, Anthropic, Meta, Amazon, Microsoft
- **Problem solutions** — Well-explained Python solutions with pattern identification
- **AI Engineering content** — RAG, fine-tuning, MLOps, cloud AI services
- **GCP guides** — Vertex AI, BigQuery, Cloud Run, real customer scenarios
- **Corrections** — Anything wrong, outdated, or unclear

## How to Contribute

### Option 1: Quick fix (typo, broken link, small correction)

1. Click "Edit this file" on GitHub
2. Make your change
3. Submit a pull request with a clear description

### Option 2: Adding content (new problem, new section)

1. Fork the repository
2. Create a branch: `git checkout -b add/topic-name`
3. Make your changes following the style guide below
4. Submit a pull request using the PR template

### Option 3: Share your interview experience

Use the **"Add Company Experience"** issue template. No pull request needed — just fill out the form.

## Style Guide

### Markdown formatting

- Use ATX-style headers (`##`, `###`)
- Tables for comparison data
- Code blocks with language tags (` ```python `)
- ASCII diagrams where visuals help

### Code examples

- Python only
- Include comments explaining the WHY not just the WHAT
- Show time and space complexity
- Run your code before submitting

### Content depth

Match the depth of existing files. We're building a world-class resource, not a quick cheat sheet. If a pattern guide exists at 200 lines, your addition should be comparable.

### Tone

- **Resource sections (01-06):** Authoritative, comprehensive, professional
- **Examples and tips:** Practical, direct, no fluff

## What We Don't Accept

- Plagiarized content from LeetCode or other resources
- Solutions without explanation
- Company-specific content that violates NDAs
- Self-promotion or spam links

## Recognition

All contributors are recognized in commit history. If you make a substantial contribution, open an issue to request being added to a contributors section we're building.

## Questions?

Open an issue with the `question` label. We try to respond within 48 hours.
